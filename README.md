# 通辽麻将服务器版

这个版本已经从“四家同屏测试台”改成了可部署的房间制版本：

- 一个玩家打开网页后创建房间
- 把 6 位房间号发给其他玩家
- 其他玩家在自己的设备上输入房间号加入
- 四人到齐后，房主点击“开始游戏”
- 每个设备只显示自己的手牌，其他玩家显示正常对局视角

## 项目位置

- 原项目备份：`D:\project\tongliao-mazhang-webgame-backup-20260613-1`
- 改造后的服务器版：`C:\Users\13630\tongliao-mazhang-webgame-server`

## 运行要求

- Node.js 20+

## 本地构建

```bash
npm install
npm run build
```

## 启动方式

构建完成后只需要启动一个服务：

```bash
npm run -w @tlmz/server start
```

默认监听：

```text
http://0.0.0.0:8787
```

浏览器访问：

```text
http://你的服务器IP:8787
```

前端静态页面和 WebSocket 都由同一个 Node 服务提供，不再需要单独启动 `vite`。

## 部署步骤

1. 把 `C:\Users\13630\tongliao-mazhang-webgame-server` 整个目录上传到服务器。
2. 在服务器上执行：

```bash
npm install
npm run build
npm run -w @tlmz/server start
```

3. 放行服务器的 `8787` 端口。
4. 用浏览器打开 `http://服务器IP:8787`。

如果你要长期运行，建议再用 `pm2`、`systemd` 或 Docker 托管这个 Node 进程。

## 当前行为

- 创建房间后会生成 6 位房间号
- 只能加入已存在的房间
- 一个浏览器连接只占一个座位
- 房间满 4 人后才能开始
- 其他玩家手牌默认隐藏

## 说明

这次改造主要解决的是“像正常游戏那样通过房间号多人联机”和“一个地址直接部署访问”。

规则层仍然是你原项目里的那套实现，因此如果后面你还要继续做：
